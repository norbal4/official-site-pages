# official-site-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常是与特定站点相关的静态资源，以独立文件形式进行版本管理和部署。仓库本身不对应任何特定域名或网站，仅作为页面文件的存储与分发中心。

## 目录结构说明

```
├── README.md          # 本文件
├── pages/             # 存放所有 HTML 页面文件
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 可选，存放页面引用的公共资源（如 CSS、JS、图片）
    └── ...
```

- `pages/`：核心目录，每个 HTML 文件对应一个独立页面。
- `assets/`：可选目录，用于存放页面间共享的静态资源文件。

## 页面归档说明

每个 HTML 页面均为独立、完整的静态文件，不依赖外部服务或数据库。页面内容与样式均内嵌或通过本地资源引用。归档时请遵循以下规范：

- 文件名使用小写英文，单词间以连字符 `-` 分隔。
- 页面文件放置于 `pages/` 目录下，避免嵌套过深。
- 如需引用 `assets/` 中的资源，请使用相对路径。

## 维护说明

- 新增页面：将新 HTML 文件放入 `pages/` 目录，并确保其自包含性。
- 更新页面：直接替换对应文件，保留历史版本可通过 Git 追溯。
- 删除页面：移除对应文件，并检查是否有其他页面引用其资源。
- 资源管理：若页面使用公共资源，请将资源放入 `assets/` 并保持路径一致性。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务将本仓库内容部署为可访问的站点。具体部署方式请参考相应平台的文档。
